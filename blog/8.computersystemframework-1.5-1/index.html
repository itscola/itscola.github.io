计算机系统结构系列

一 并行性

现代以及未来的计算机，都是并行的。并行能大大提高计算效率。

1. 并行性的定义

计算机在 同一时刻 或 同一时间间隔，进行运算或操作的特性，称为并行性。

按照 同一时刻 或 同一时间间隔 可将 并行性 分类两类（两重含义）

同时性 和 并发性。

1.1 同时性

定义：一个以上的事件 同一时刻 发生。

如CPU的两个核，分别同时运行两个程序，这就是同时发生的。

1.2 并发性

定义：一个以上的事件 同一时间间隔 发生。

如很多游戏，多个人在一起PVP（Player vs player），实际上是 一个人动一下 之后下一个人动一下 再 下一个人动一下….. 这就是并行性。

由于切换非常快，所以就看上去是同时在动。

2. 并行性目的

并行性的目的是为了能并行处理，以提高计算机解题效率。

并发和并行（补充）

和性质命名不太一样。我们说的“并行”“并发”是两种方式 ，是有区别的。

并行 对应的 同时性。
并发 对应的是 并行性。

不要把 性质 和 方式 搞混。

二 并行性等级划分

并行性可以以多种角度进行多种划分。

1. 计算机系统执行程序的角度

从计算机系统执行程序的角度来看，并行性等级由低到高可分为四级。

1. 指令内部

一条指令内部各个微操作之间的并行执行。

传统机器语言机器级 用 微程序 解释一条指令，然后会有多个 微操作（步骤） 交给 微程序机器级 由硬件执行。这些微操作可以并行执行。

一条指令 -> 多个微操作（并行）

计算机系统层次结构

2. 指令之间

多条指令的并行执行。

我们一条指令 对应的 多个微操作（步骤）可以并行，我们的指令之间，也可以并行。

多条指令（并行） -> 多个微操作（并行）

3. 任务或进程之间

多个任务或程序段的并行操作。

  1. 任务 指 进程，有时也可以指线程。
    注意，进程和线程是有区别的。
    操作系统的不同任务之间是进程，而进程的子任务是线程。

new Thread(()->{}).start()
这是一段java代码，它创建了一个新线程去执行{}内的代码（任务）。就是这个线程。 它们的并行性 就是指的 多线程。 放大到进程也是一样。 **任务可以有并行性。 **

  1. 程序段 指的是一段程序，比如代码中调用的一个函数（方法）。 这些也可以有并行性。 同时 或 同一时间间隔 调用多个函数（方法）。

5. 作业或程序之间

多个作业或多道程序的并行执行。

多个作业可以理解为，我一个计算机进行作业不够，那我再来一个，再不够再来一个。 就让多个计算机并行作业。

多个计算机跑一个程序协同工作，并行计算。

多道程序就更好理解了，计算机中几个独立的程序并行执行。

2. 计算机处理数据的角度

从计算机系统处理数据的角度来看，并行性等级也由低到高可分为四级。

0. 字，比特，字节

这里需要一个铺垫，作为以下几种方式的基础。首先要知道，什么是 字(word)、比特(bit)、字节(Byte)。

  1. 位 是计算机中最基本的组成单位，它可以用一个0或一个1表示，指的就是 比特（bit）。

我们都知道二进制，都知道计算机只认识0或1。实际上，0或1只是用来表达的方式，计算机连0或1都不认识。计算机只认识有电和没电两种状态。计算机是可以做到16进制等的，只是完全没必要，目前二进制就够用了，没必要再去无意义的增加成本了。

  1. 字节（Byte），一个字节（Byte）等于八个比特（bit）。八个比特（bit）能表示2^8=256种情况。

  2. 字（word）有几个字节（Byte）表示，

16位计算机中 1字（word）= 2字节（Byte）= 16位（bit）

32位计算机中 1字（word）= 4字节（Byte）= 32位（bit）

64位计算机中 1字（word）= 8字节（Byte）= 64位（bit）

所以现在你知道多少位计算机是怎么来的了。

字节是计算机处理数据中，最基本的单位。

因为字节不会改变，永远1=8。

网络运营商为了让网络看着快，说100兆宽带，300兆宽带，500兆宽带。比如300兆宽带，实际上说的是 300Mb/s 而不是 300MB/s。所以办的宽带速度换算成实际速度，要除以8。

300Mb/s -> 300/ 8 -> 37.5MB/s

懂了吗？所以你的300兆的网络，下载速度最高也大约就37.5MB/s

1. 位串字串

同时只对一个字的一位进行处理，这通常是指传统的串行单处理机，没有并行性。
（每次只对一个字的一位进行处理）

就是每次对一个字（word）的一个位（bit）进行处理。

效率不太行。

2. 位并字串

同时对一个字的全部位进行处理，这通常是指传统的并行单处理机，开始出现并行性。

就是每次对一个字（word）的 所有位（bit）并行 进行处理。

效率提高点了

3. 位片串字并

同时对许多字的同一位（称位片）进行处理，这通常是指 传统的并行单处理机。

就是，字（word）并行处理，每个字的同一位（bit）（位片）串行处理。

谁起的位片这个名字啊

4. 全并行￼

同时对许多字的全部或部分位组进行处理。

字（word）并行，位（bit）也并行。￼

3. 计算机信息加工的各个步骤和阶段的角度

1. 存储器操作并行

在一次存储访问中（周期）可以一些方式访问多个字。 如 单体单字、多体单字、多体多字。

我们可以按内容访问方式，对内容中大量的字，以 位片串字并 或 全并行方式 ，进行比较，检索，更新，变换等操作

简而言之，就是每次对储存访问用到的操作，可以对字并行处理。

典型的例子是 并行储存器系统 和 以相联储存器微核心构成的 相联处理机。

2. 处理器操作步骤并行

指令的取指，分析，执行，浮点加法的求阶差、对阶、尾加、舍入、规格化等操作，执行步骤在时间上重叠流水地进行。

典型例子是 流水线处理机。

看完你可能会说 “什么玩意？？”

简单来说，就是对指令的各种操作，以流水线的方式处理。

比如 A负责处理取指，B负责处理分析，C负责处理执行。
那么要一个command（指令）过去，其实是A处理完取指，交给B处理分析，再交给C处理执行。

3. 处理器操作并行

通过重复设置大量处理单元。
让它们在同一控制器下按同一指令要求对 向量 、数组中各元素同时操作。

我们在本篇文章上方（1.指令内部）说过，一条指令，有多个步骤（微操作）。

这里，就是用多个处理单元，通过同一控制器，对一条指令要求的步骤同时处理。

典型例子是，阵列处理机。

就是说，多个处理机，对一个指令处理，对一个指令的多个步骤（微操作）并行处理。属于 单指令流多数据流计算机。

4. 指令、任务、作业并行

这是较高级的并行。

指令级以上的并行（就这个）是多个处理机同时对多条指令和相关的多数据组进行处理。

典型例子是“多处理机”

就是说，多个处理机，对多个指令并行处理，对每个指令的多个步骤（微操作）并行处理。属于 多指令流多数据流计算机。
