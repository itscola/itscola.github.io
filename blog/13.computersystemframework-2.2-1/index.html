1. 寻址方式(addressing mode)的定义⭐️

寻址方式指的是 指令通过什么方式来 寻找/访问 需要的数据(操作数或信息)的地址。
不同的计算机系统都有不同的寻址方式。

也就是说，执行一条指令， 这条指令需要通过什么方式来访问数据的 地址。

地址在指令中主要有两种：

1. 形式地址：通常需要给指令的每一个操作数加一个地址。

先要知道一条指令由 操作码 和 操作数 构成。

操作数指出了指令执行的操作所需要数据的来源。

图中操作数的位置/字段 可以是操作数本身。
也可以是 要指向操作数 的操作地址，也可以是 要指向操作数 的操作地址 的计算方法。也就是地址码。

而操作码 就表示我们要进行的操作。 比如ADD ，MOV …….

一条汇编指令：MOV(操作码) 操作数字段(操作数 操作数)
如 MOV AH，06H

每个指令里都可能包含多个地址，也可能不含有地址。但如果指令里给出了地址，那么我们称 指令中给出的地址为 形式地址。

如 ADD A1 , A2 这里的A1和A2都是形式地址。

2. 实际地址：使用形式地址信息按照一定规则计算出来的 数值/数据 称为实际地址

通常我们指令中给出的形式地址，需要按照规则进行一定的计算然后才能得出真正的 数值/数据 ， 我们把这通过计算后的 真正的 数值/数据 称为 实际地址。

实际地址才能访问 寄存器 或 主存 的数据。

如 ADD A1 , A2 + MEM[1200] ， 操作数字段的第二个操作数(A2 + MEM[1200]) 需要先进行计算, 然后形成的实际访问 寄存器 或 主存 中 真正数据的地址 就是实际地址。之后，赋值给A1时，再形成实际地址。

(通常在指令中看不到)

形式地址和实际地址有可能一样，也可能不一样。 寄存器通常一样，存储器中通常不一样。

2. 寻址方式分类⭐️

1. 面向寄存器寻址

寄存器是CPU内部用来存放数据的一些小型存储区域，用来暂时存放参与运算的数据和运算结果。

面向寄存器寻址：操作数可以取自寄存器和主存，结果通常放在寄存器中，少部分放在内存。

2. 面向堆栈寻址

堆栈是一个特定的存储区或寄存器，是个先进后出的队列。
如需了解堆栈，建议搜索java堆栈。

面向堆栈寻址：主要访问堆栈，少量访问主存或寄存器。

3. 面向主存寻址

主要访问主存，少量访问寄存器。

4. 补充

还有一种面向IO寻址，通常已经包括在主存那边了。
通常是多个寻址方式同时使用，而不是只是一个。

4. 寻址方式在指令中的指明⭐️

4-1. 操作数 和 地址码 和 操作码 的区别【补充】

我们前面说过， 一条指令的格式 由
操作码 + 操作数 来构成。

定义 解释
操作码: 指令执行的方式
操作数: 指令执行的对象
地址码: 操作数这个对象的内容

我们可以这样理解。
操作数是一个盒子，而盒子里面装的是 地址码。

所以 我们也可以说 一条指令的结构是 ： 操作码 + 地址码

1. 寻址方式在指令中的指明的概念

占用操作码的某些位置，来指明寻址方式。
也就是 占用操作码的一些位置，来指明指令寻址的方式。比如直接寻址，或是 间接寻址

我们需要在指令中的 操作码 和 地址码 之间选择一个占用，来表达寻址方式。
这也就是两种指明方式，一种是 占用操作码 ，另一种是占用 地址码。

寻址方式在指令中一般有两种不同的指明方式，不同计算机可用其中的一种。一种方式是占用操作码种的某些位来指明。另一种方式是不占用操作码，而是在地址码部分专门设置寻址方式位字段指明。 【下方解释】

2. 占用操作码 指明方式

实际上是增加多个 操作码(指令) 实现。

比如
R1 = R1 + R2 -> ADD R1 , R2

如果我要再实现
R1 = R2 + 20
就可以加一条指令 假如加一个 ADDN 那么就是 ADDN R2 , 20

这就是占用操作码来实现指明寻址方式了。

3. 占用地址码 指明方式

在地址码专门设置寻址方式位字段指明。

就是说，每一个地址码那边，加上一个“地址标志符”，来表示寻址方式。

假如 R1 的地址是 10011 （）
我可以给它的地址码加上点内容 来表示 寻址方式。 比如在前面加个 111 那就变成了 11110011。表示直接寻址。

那么可以加个101 变为 10110011 表示间接寻址。

如果这个地址标识符是两位 如10 那就4种寻址方式。 3位 那就支持8种方式。

4. 两种方式比较

相比而言，寻址方式位(占用地址码)的寻址灵活，操作码短，但学要专门的寻址方式位字段。

就操作码 和 寻址方式位 的总数来看，可能会比占用操作码种某些来指明的方式更长。（需要占用的位更多 就如 “3. 占用地址码 指明方式” 所示。）
