软件实现和硬件实现的比例关系

通常，同一个功能，我们既可以用软件实现也可以用硬件来实现。

用软件来实现更灵活，更强大，设计和重复生产费用更低。但效率远不如硬件实现。
用硬件实现效率更高，性能更高，但设计和生产费用较为高昂，灵活性不如软件。
（通常不存在只用软件实现或只用硬件实现的类计算机设备）

通过这篇文章，你将了解到，计算机 手机 设备 等设计 软硬件取舍，以及设备产量与硬件费用的关系。
（生产量越多，设备在一定资金下，能用的硬件越多？）

生产软硬件分别需要支付的费用

生产软硬件 都需要 研发费用(Design) 和 生产费用(Manufacture)。
软件通常需要多次设计(研发)，硬件通常只需要设计一次。

我们设硬件每次的研发费用为D(hardware) 简称 D(h). 同时设每次重复生产费用为M(h)。
设软件研发每次的研发费用为 D(software) 简称 D(s). 同时设每次重复生产费用为 M(s)。

D(h) 约等于 100D(s). 每研发一个硬件的费用 大概为 研发100个软件的费用。
M(h)约等于 100M(h). 每生产一个硬件的费用 大概为 生产100个软件的费用。
(可以说硬件和软件比 是非常贵的。)

实际生产费用

我们设生产了V个设备，同时设研发了C次。
设一个功能在一台设备上出现了R次。

每台设备的软件生产费用

那么每台设备 软件实际生产费用为
C * D(s) / V + R * M(s).
指的是
研发次数 * 研发软件费用 / 设备生产数 + 功能在设备上出现次数 * 软件重复生产费用

每台设备的硬件生产费用

那么每台设备 硬件实际生产费用为
D(h)/v + M(h)
指的是
研发硬件费用 / 设备生产数 + 硬件重复制造费用.
(硬件通常只需要研发一次，每台设备上 硬件重复出现通常也一次 因此只需要制造一次)

每台设备软件生产费用和硬件生产费用的关系

我们假设一个功能实现最省钱的情况 我们尽可能使用 硬件(性能极高)。
当 我们 每台设备的硬件费用 小于 每台设备的软件费用 此时 用硬件最为合适。
也就是说 D(h)/v + M(h) < C * D(s) / V + R * M(s) 时 此功能用硬件最合适。

那么我们把通常的情况代入
使 D(h) 约等于 100D(s) .
使 M(h)约等于 100M(h) .

得到:
100D(s)/v + 100M(s) < C * D(s) / V + R * M(s).
所以，C和R要足够大，才能使等式成立。
也就是说 一个功能 经常用到 经常出现 才适合用硬件。

我们可以假设 D(s) = 10^4 M(s) 设计费远大于软件的出现费用。

得到:
10^6 * M(s)/v + 100M(s) < C * 10^4 * M(s) / V + R * M(s).
M(s)一定为正数，所以我们两边同时除以M(s)。

得到:
10^6 / v + 100 < C * 10^4 / V + R.

C 一般小于 100. + 100 和 R 较小 我们可以在这忽视。

那么有 10^6 / v < C * 10^4 / V .

可以看到 只有V的量越大 这个不等式才越接近。

也就是说 当设备的产量越大 用硬件的部分在划算的情况下 就可以越多。
(即使不等式不会成立)

总结

  1. 也就是说 一个功能 经常用到 经常出现 才更适合用硬件。
  2. 当我们要生产的设备产量越大，我们用硬件的比例就可以越多(越适合)。
  3. 并不是用硬件实现功能的比列越大越好。这还要考虑资金等问题。
